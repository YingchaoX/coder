# Coder 需求说明（代码对齐基线）

## 1. 文档定位
本目录描述**当前仓库实现**对应的产品行为基线（以代码为准），用于：
- 和实现逐条对齐，避免“文档写了但代码未落地”。
- 为后续迭代提供稳定验收口径。

基线版本：`2026-02-11`（仓库当前 `main` 工作区）。

## 3. 与设计文档关系
- 需求文档回答“用户能看到什么、系统会做什么”。
- 设计文档回答“代码如何实现、模块如何协作”。

设计文档入口：`docs/design-v2.md`
